JavaScript dialogs are fairly common in web applications. The best way to automate these is to override the JavaScript functions to return the value you want, so when they’re meant to show, they don’t! [sourcecode language="ruby" light="true"] # don’t return anything for alert browser.execute_script(“window.alert = function() {}”) # return some string for prompt to simulate user entering it browser.execute_script(“window.prompt = function() {return ‘my name’}”) # return null for prompt to simulate clicking Cancel browser.execute_script(“window.prompt = function() {return null}”) # return true for confirm to simulate clicking OK browser.execute_script(“window.confirm = function() {return true}”) # return false for confirm to simulate clicking Cancel browser.execute_script(“window.confirm = function() {return false}”) [/sourcecode]
